# 工作流指南

## 创建日程标准流程

1. **确认当前日期** — 使用 `session_status` 或 `date` 命令获取当前时间
2. **解析用户的时间描述** — 将"今天"、"明天"、"下周"等转换为具体日期
3. **确认完整时间信息** — 与用户确认会议标题、时间、参与者等
4. **执行创建** — 调用脚本创建日程
5. **⚠️ 必须添加参与者** — 创建后必须将用户添加为参与者，否则日程不会显示在用户日历中

**为什么必须添加参与者**：企业微信 API 创建的日程默认在应用日历里，**不会自动显示在个人日历中**。必须将用户添加为参与者后，日程才会同步到用户的日程视图中。

## 修改日程标准流程

当用户要求**修改、改期、延期、调整**已有日程时，使用**更新接口**：

**识别关键词**：把...改成...、改到...、延期到...、调整到...、时间改成...、地点改成...、内容改成...

**处理流程**：
1. **获取日程ID** — 通过 `list-user` 查询用户的日程列表，找到要修改的日程
2. **确认修改内容** — 与用户确认新的时间、标题、地点等
3. **执行更新** — 使用 `update` 命令修改日程
4. **保持参与者** — 使用 `--skip_attendees 1` 保持原有参与者不变

**注意事项**：
- 更新接口必须提供 `--schedule_id`、`--start`、`--end`
- 使用 `--skip_attendees 1` 保持参与者不变（推荐）
- 如需修改参与者，使用 `add-attendees` / `del-attendees` 接口
- 已预约会议室的日程无法修改时间（需先取消会议室）

## 私聊场景处理

### 创建会议

在私聊中处理日程请求时，**无法通过@提取参与者**，需要通过**语义匹配 USER.md 中的联系人**来确定参与者。

**识别关键词**：明天/下周/周五... 开会/会议/评审...、提醒我... 时间... 事情、创建/添加/设置... 日程/会议/提醒、和 XXX 开会等包含人名的表达

**处理流程**：
1. **解析时间描述**（今天/明天/下周等）
2. **提取会议标题和事项**
3. **检测部门关键词**：如果提到"运营部"、"运营"等 → **自动添加运营部成员** `${USER_OPERATIONS}`
4. **匹配参与者**：读取 `USER.md` 中的联系人列表
   - 直接匹配姓名（如 `${USER_ADMIN}`、"贞总"→`${USER_ADMIN}`）
   - 匹配部门（提到"运营部"→自动添加`${USER_OPERATIONS}`）
5. **确认参与者**：向用户确认识别到的参与者是否正确
6. **创建日程**：
   - **attendees**：当前私聊用户 + 匹配到的联系人 + 部门相关人员
   - **admins**：当前私聊用户

**私聊场景示例**：

| 场景 | 用户说法 | 参与者 |
|------|---------|--------|
| 与联系人开会 | "明天下午3点和贞总开个评审会" | 当前用户 + ${USER_ADMIN} |
| 纯个人提醒 | "提醒我明天早上9点吃饭" | 仅当前用户 |
| 涉及运营部 | "约运营部明天下午开会" | 当前用户 + ${USER_OPERATIONS} |
| 混合场景 | "约${USER_PARTICIPANT_A}和运营部明天下午开会" | 当前用户 + ${USER_PARTICIPANT_A} + ${USER_OPERATIONS} |

### 查询日程

**识别关键词**：明天/今天/下周... 有什么... 事情/安排/日程/会议、查看... 日程/安排、提醒... 今天/明天... 做什么

**处理流程**：
1. 确认当前日期和时间
2. 解析用户询问的时间范围
3. 调用 `list-user` 获取日程列表
4. 筛选出包含该用户的日程
5. 整理并回复用户

```bash
./wecom-schedule.sh list-user "user1" "$DATE_TOMORROW" "$DATE_TOMORROW"
```

## 群聊场景处理

在群聊中，如果用户@了其他人并同时@我：

**处理流程**：
1. **提取被@用户** — 从消息中识别被@的用户ID
2. **发起人** — 发送消息的用户（添加到参与者 + 设为管理员）
3. **检测部门关键词** — 提到"运营部"自动添加 `${USER_OPERATIONS}`（相关部门成员）
4. **组织者** — 机器人（应用自身）

**群聊示例**：

| 场景 | 命令 |
|------|------|
| 普通会议 | `--attendees "$CREATOR_USERID,${USER_X},${USER_Y}"` |
| 涉及运营部 | `--attendees "$CREATOR_USERID,${USER_OPERATIONS}"` |
| 查询发起人日程 | `list-user "$CREATOR_USERID" "$DATE_TOMORROW" "$DATE_TOMORROW"` |

**注意事项**：
- 群聊中需要开启 `groupPolicy` 才能响应
- 被@的用户ID需要从消息元数据中提取
- **查询时只返回发起人的日程**，保护群成员隐私
